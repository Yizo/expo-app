# react-native-svg — 在 Expo / React Native 中使用 SVG

## 文档解决的问题

在 Web 开发中，SVG 可以直接以 `<svg>` 标签内联到 HTML 中，浏览器原生渲染。但在 React Native 环境下，没有浏览器 DOM，也没有原生的 `<svg>` 标签，因此需要专门的库来提供 SVG 渲染能力。

`react-native-svg` 就是解决这个问题的库——它让你在 React Native 应用中以组件化的方式使用 SVG，并支持动画和交互。

**支持平台**：Android、iOS、macOS、tvOS、Web 以及 Expo Go。

> **对 React Web 开发者的意义**：如果你曾经在 Web 项目中用 `<svg>` 标签画过图形，这个库提供的 API 非常相似——`Circle`、`Rect`、`Path` 等组件名和属性基本对应 Web SVG 的同名元素和属性，学习成本较低。

## 阅读前需要理解的背景知识

### 什么是 Expo Go？

Expo Go 是 Expo 提供的一个沙盒应用，你可以在手机上安装它来快速预览和调试你的 Expo 项目，而不需要配置完整的原生开发环境（Xcode / Android Studio）。类似于 Web 开发中直接在浏览器打开 `localhost` 预览。

### 什么是 "Bare React Native 项目"？

Expo 项目有两种模式：

- **Managed Workflow（托管工作流）**：Expo 帮你管理原生层，你只需要写 JavaScript/TypeScript。这是推荐的新手方式。
- **Bare Workflow（裸工作流）**：你完全控制原生代码（ios/ 和 android/ 目录），类似标准的 React Native 项目。

本文档的安装命令主要针对 Managed Workflow。如果你使用 Bare Workflow，需要先确保项目中已安装 `expo` 包，再按照 `react-native-svg` 的 GitHub README 中的安装说明操作。

### `npx expo install` 与 `npm install` 的区别

在 Expo 项目中，推荐使用 `npx expo install` 而不是直接 `npm install`。原因是 `expo install` 会自动选择与当前 Expo SDK 版本兼容的包版本，避免版本冲突导致的运行时错误。这类似于 Web 开发中某些框架提供的 `add` 命令（如 `vue add`、`ng add`）。

## 安装

使用你偏好的包管理器配合 Expo CLI 安装：

```sh
# 使用 npm
npx expo install react-native-svg

# 使用 yarn
yarn expo install react-native-svg

# 使用 pnpm
pnpm expo install react-native-svg

# 使用 bun
bun expo install react-native-svg
```

> **说明**：以上四条命令效果相同，只是使用不同的包管理器。选择你项目中已经在用的那个即可。Expo CLI 会自动处理原生依赖的链接（linking），在大多数情况下你不需要手动执行 `react-native link` 或修改原生配置文件。

## API 使用

### 导入方式

```tsx
import Svg, { Circle, Rect } from 'react-native-svg';
```

**导入说明**：

- `Svg` 是根组件，相当于 Web 中的 `<svg>` 标签，作为所有 SVG 图形的容器。
- `Circle`、`Rect`、`Path`、`ClipPath`、`Polygon` 等是绘图子组件，对应 SVG 规范中的同名元素。
- 也可以用命名空间方式导入：`import * as Svg from 'react-native-svg'`，然后通过 `Svg.Circle`、`Svg.Rect` 等访问。

### 基本用法示例

```tsx
import Svg, { Circle, Rect } from 'react-native-svg';

export default function SvgComponent(props) {
  return (
    <Svg height="50%" width="50%" viewBox="0 0 100 100" {...props}>
      <Circle
        cx="50"
        cy="50"
        r="45"
        stroke="blue"
        strokeWidth="2.5"
        fill="green"
      />
      <Rect
        x="15"
        y="15"
        width="70"
        height="70"
        stroke="red"
        strokeWidth="2"
        fill="yellow"
      />
    </Svg>
  );
}
```

**代码解读**：

| 属性 | 含义 | Web SVG 对应 |
|---|---|---|
| `height` / `width` | SVG 画布的尺寸，支持百分比和像素值 | `<svg>` 的 `height` / `width` 属性 |
| `viewBox` | 定义 SVG 内部坐标系，格式为 `"minX minY width height"` | `<svg>` 的 `viewBox` 属性，含义完全一致 |
| `cx` / `cy` | 圆心坐标 | `<circle>` 的 `cx` / `cy` |
| `r` | 圆的半径 | `<circle>` 的 `r` |
| `stroke` | 描边颜色 | CSS `stroke` 属性 |
| `strokeWidth` | 描边宽度 | CSS `stroke-width`（注意 React Native 用驼峰命名） |
| `fill` | 填充颜色 | CSS `fill` 属性 |
| `x` / `y` | 矩形左上角坐标 | `<rect>` 的 `x` / `y` |

> **与 Web 的关键差异**：在 Web 中，SVG 属性可以使用短横线命名（如 `stroke-width`），但在 React Native 中统一使用驼峰命名（如 `strokeWidth`）。这与 React Web 中的 JSX 写法一致（React Web 也推荐驼峰），所以如果你熟悉 React Web 的 SVG 写法，这里几乎无缝衔接。

### 可用的绘图组件

`Svg` 根组件下支持以下绘图原语（primitives）：

- **`Circle`** — 圆形
- **`Rect`** — 矩形
- **`Path`** — 路径（用于绘制任意形状，如贝塞尔曲线）
- **`ClipPath`** — 裁剪路径
- **`Polygon`** — 多边形

这些组件覆盖了 SVG 规范中的大部分标准元素和属性。完整的 API 细节请参考 [react-native-svg GitHub 仓库](https://github.com/software-mansion/react-native-svg)。

## 实用技巧

文档提供了几个在实际开发中非常有用的工具和技巧：

### 1. 图标资源获取 — Lucide

[Lucide](https://lucide.dev/) 是一个开源图标库，提供大量高质量的 SVG 图标。在 Web 开发中你可能已经用过 `lucide-react`，在 React Native 中也有对应的包 `lucide-react-native`，可以直接以组件方式使用这些图标。

### 2. 设计与编辑 — Figma

[Figma](https://www.figma.com/) 是免费的矢量设计工具，可以用来设计或编辑 SVG 图形。设计师在 Figma 中完成设计后，可以导出 SVG 代码，再通过下面的工具转换为 React 组件。

### 3. 文件体积优化 — SVGOMG

[SVGOMG](https://jakearchibald.github.io/svgomg/) 是一个在线 SVG 压缩工具，可以去除 SVG 文件中的冗余信息（如编辑器元数据、注释等），显著减小文件体积。

> **重要注意**：压缩 SVG 时，务必保留 `viewBox` 属性。在 Android 平台上，缺少 `viewBox` 会导致 SVG 渲染异常或性能问题。SVGOMG 默认可能会移除它，需要在设置中手动勾选保留。

### 4. SVG 转 React 组件 — SVGR

[SVGR](https://react-svgr.com/) 是一个浏览器工具，可以将 SVG 文件直接转换为 React 组件代码。在 Web 开发中你可能用过 `@svgr/webpack` 或 `vite-plugin-svgr`，SVGR 的在线工具提供了同样的能力，生成的组件代码可以直接在 React Native 项目中使用（前提是导入来自 `react-native-svg` 而非 DOM 元素）。

## 注意事项、限制条件和坑点

### viewBox 属性在 Android 上的重要性

`viewBox` 不仅仅是一个坐标系定义工具——在 Android 平台上，它对渲染性能有直接影响。缺少 `viewBox` 可能导致：

- SVG 图形无法正确缩放
- 渲染性能下降
- 图形显示位置偏移

**建议**：始终为 `<Svg>` 组件指定 `viewBox`，即使你已经设置了 `width` 和 `height`。

### 属性命名风格

React Native 中的 SVG 属性使用驼峰命名（如 `strokeWidth`、`fillOpacity`），这与原生 SVG 规范中的短横线命名（`stroke-width`、`fill-opacity`）不同。如果你从 Web 项目中复制 SVG 代码到 React Native，需要手动转换属性名。

### 不支持所有 SVG 特性

`react-native-svg` 覆盖了大部分常用 SVG 元素和属性，但并非 100% 实现 SVG 规范。一些高级特性（如部分滤镜效果 `filter`、某些渐变用法）可能不支持或行为与 Web 不同。在使用复杂 SVG 之前，建议先在目标平台上测试验证。

### 从 Web 迁移 SVG 代码时的差异

| Web 中的写法 | React Native 中的写法 | 说明 |
|---|---|---|
| `<svg>` | `<Svg>` | 组件名大写 |
| `<circle>` | `<Circle>` | 所有元素都是大写开头的组件 |
| `stroke-width="2"` | `strokeWidth="2"` 或 `strokeWidth={2}` | 属性名驼峰化 |
| `style="fill: red"` | `style={{ fill: 'red' }}` 或直接 `fill="red"` | 样式使用对象语法，或直接用属性 |
| 内联在 HTML 中 | 作为独立组件使用 | SVG 必须是组件树的节点 |

## React Web 开发者需要特别注意的地方

1. **没有内联 SVG**：在 Web 中你可以直接在 JSX 里写 `<svg><circle ... /></svg>`。在 React Native 中，必须从 `react-native-svg` 导入组件，不能使用 HTML 原生标签。

2. **样式系统不同**：Web 中 SVG 可以通过 CSS 类名、CSS 变量、外部样式表等方式控制样式。React Native 中没有 CSS 级联，SVG 的样式主要通过组件属性（props）直接传入。

3. **尺寸单位**：Web 中 SVG 的 `width` / `height` 可以使用 `em`、`rem`、`vh` 等 CSS 单位。React Native 中主要使用像素值（数字）或百分比字符串。

4. **事件处理**：`react-native-svg` 支持交互（如点击事件），但事件对象的属性和 Web 的 DOM 事件不同，使用的是 React Native 的手势响应系统。

## 实际开发建议

1. **优先使用图标库**：对于常见的图标需求（箭头、菜单、关闭按钮等），直接使用 `lucide-react-native` 或 `@expo/vector-icons`，比自己导入 SVG 文件更高效。

2. **复杂图形使用 SVGR 转换**：对于设计师提供的复杂 SVG 文件，使用 SVGR 转换为组件，而不是手动逐行翻译代码。转换后记得检查导入来源是否为 `react-native-svg`。

3. **始终指定 viewBox**：这是一个容易忽略但对 Android 平台影响很大的属性，养成始终设置的习惯。

4. **压缩 SVG 时保留关键属性**：使用 SVGOMG 优化时，确认 `viewBox` 未被移除。

5. **多平台测试**：SVG 在 iOS 和 Android 上的渲染可能存在细微差异，尤其是涉及渐变、阴影、裁剪等高级效果时，务必在两个平台上都进行测试。

## 总结

`react-native-svg` 是 Expo / React Native 生态中使用 SVG 的标准方案。它提供了与 Web SVG 规范高度一致的组件 API，让熟悉 React Web 的开发者可以较低成本地在移动端使用矢量图形。核心要点包括：通过 `npx expo install` 安装以确保版本兼容、使用大写组件名和驼峰属性名、始终指定 `viewBox` 以保障 Android 平台表现、以及借助 Lucide / Figma / SVGOMG / SVGR 等工具提升开发效率。
