# Row：通用水平布局容器

`Row` 是 `@expo/ui` 提供的水平布局组件，用于将多个子元素从起始端到结束端依次排列。

> **版本提示：**原文档属于下一个 Expo SDK 版本的未发布文档。当前稳定版本是 SDK 56；实际开发时应以对应 SDK 版本的文档为准。

## 文档解决的问题

这篇文档主要说明如何：

- 使用 `Row` 创建水平排列的界面。
- 设置子元素之间的水平间距。
- 控制子元素在垂直方向上的对齐方式。
- 配合 `Spacer` 将内容推向容器两端。
- 使用 `Row` 的交互、生命周期、样式和测试属性。

适合工具栏、操作栏、标签组、水平表单项，以及“左侧内容 + 右侧内容”等布局场景。

## 平台实现与核心概念

`Row` 支持 Android、iOS 和 Web，并包含在 Expo Go 中。

虽然业务代码统一使用 `Row`，但各平台的底层实现不同：

| 平台 | 底层实现 |
| --- | --- |
| Android | Jetpack Compose 的 `Row` |
| iOS | SwiftUI 的 `HStack` |
| Web | 使用 Flex 布局的 `View` |

Jetpack Compose 和 SwiftUI 分别是 Android 与 iOS 的原生声明式 UI 框架。可以把它们类比为原生平台上的组件化 UI 系统，但它们不是浏览器 DOM，也不使用 CSS。

`@expo/ui` 在三端提供统一的 React API，并将部分属性转换为对应平台的原生布局配置。

### 与 React Web Flexbox 的对应关系

对于 React Web 开发者，可以近似地把 `Row` 理解为：

```css
display: flex;
flex-direction: row;
```

但这只是概念类比。Android 和 iOS 上的 `Row` 并不是 HTML 元素，也不会直接执行 CSS。

`Row` 包含两个方向：

- **主轴：**水平方向，子元素从起始端向结束端排列。
- **交叉轴：**垂直方向，由 `alignment` 控制。

## 安装

根据项目使用的包管理器执行对应命令：

```sh
# npm
npx expo install @expo/ui

# yarn
yarn expo install @expo/ui

# pnpm
pnpm expo install @expo/ui

# bun
bun expo install @expo/ui
```

`expo install` 与普通的 `npm install` 不完全相同。它会结合当前 Expo SDK，选择兼容的软件包版本，通常更适合安装 Expo 模块。

如果是在已有的普通 React Native 工程中使用，而不是由 Expo 创建或管理的项目，需要先按照 Expo Modules 的安装流程将 `expo` 接入工程。

## 基础用法

```tsx
import { Host, Row, Text } from '@expo/ui';

export default function RowExample() {
  return (
    <Host matchContents>
      <Row spacing={8}>
        <Text>One</Text>
        <Text>Two</Text>
        <Text>Three</Text>
      </Row>
    </Host>
  );
}
```

这个示例完成了三件事：

1. `Host` 为通用 `@expo/ui` 组件提供承载环境。
2. `Row` 将三个 `Text` 水平排列。
3. `spacing={8}` 在相邻子元素之间加入 8 个密度无关像素的间距。

`matchContents` 出现在示例的 `Host` 上，不属于 `Row` API。当前文档没有进一步解释该属性。

## 垂直对齐

使用 `alignment` 控制子元素在交叉轴，也就是垂直方向上的位置：

```tsx
import { Host, Row, Text } from '@expo/ui';

export default function RowAlignmentExample() {
  return (
    <Host style={{ flex: 1 }}>
      <Row spacing={8} alignment="center">
        <Text>Centered</Text>
        <Text>Centered</Text>
      </Row>
    </Host>
  );
}
```

`alignment` 支持：

| 值 | 含义 |
| --- | --- |
| `'start'` | 在垂直轴起始位置对齐，也是默认值 |
| `'center'` | 垂直居中 |
| `'end'` | 在垂直轴结束位置对齐 |

这里的 `start` 和 `end` 控制的是垂直交叉轴，不是水平方向上的左右对齐。

> **React Web 易错点：**不要把 `alignment` 直接理解成 CSS 的 `justify-content`。在水平布局中，它更接近 `align-items`。

## 使用 Spacer 分隔内容

需要把两个元素推向容器两端时，可以在中间加入可伸缩的 `Spacer`：

```tsx
import { Host, Row, Text, Spacer } from '@expo/ui';

export default function RowSpacerExample() {
  return (
    <Host style={{ flex: 1 }}>
      <Row>
        <Text>Leading</Text>
        <Spacer flexible />
        <Text>Trailing</Text>
      </Row>
    </Host>
  );
}
```

`Spacer flexible` 会占用可用的剩余空间，因此：

- `Leading` 留在起始端。
- `Trailing` 被推向结束端。

这在效果上类似于 Web Flexbox 中让中间元素增长并占满剩余空间，例如 `flex-grow: 1`。

**基于文档内容推导：**要产生明显的“两端分布”效果，`Row` 必须拥有可供 `Spacer` 扩展的剩余宽度。如果容器宽度只与内容相同，`Spacer` 就没有足够空间将两侧内容拉开。

## API

导入方式：

```tsx
import { Row } from '@expo/ui';
```

组件类型为带有 `RowProps` 的 React 元素。其主要职责是将 `children` 从起始端到结束端水平排列。

## 属性说明

### `alignment`

```ts
'start' | 'center' | 'end'
```

- 可选。
- 默认值为 `'start'`。
- 支持 Android、iOS 和 Web。
- 控制子元素在垂直交叉轴上的对齐。

### `children`

```ts
ReactNode
```

- 可选。
- 支持 Android、iOS 和 Web。
- 表示需要放入水平容器中的内容。

它可以是一个或多个 React 节点，例如 `Text`、`Spacer` 或其他兼容组件。

### `spacing`

```ts
number
```

- 可选。
- 支持 Android、iOS 和 Web。
- 设置相邻子元素之间的水平距离。
- 单位为密度无关像素。

密度无关像素用于减少不同屏幕像素密度造成的视觉尺寸差异。它不应被简单理解为设备的物理像素。

### `style`

`style` 只接受文档列出的平台无关属性：

```ts
padding
paddingHorizontal
paddingVertical
paddingTop
paddingBottom
paddingLeft
paddingRight
backgroundColor
borderRadius
borderWidth
borderColor
opacity
width
height
```

这些样式在 iOS 上会转换为 SwiftUI modifier，在 Android 上会转换为 Jetpack Compose modifier。

> **重要限制：**这不是完整的 React Native `ViewStyle`。不能因为属性名是 `style`，就假设所有 React Native 样式或 Web CSS 属性都可以使用。

尤其需要注意，文档列出的 `style` 类型中没有 `flex`、`margin`、`gap`、`justifyContent` 和 `alignItems`。子元素间距应使用 `spacing`，垂直对齐应使用 `alignment`。

### `disabled`

```ts
boolean
```

- 可选。
- 支持 Android、iOS 和 Web。
- 禁用后，组件不再响应用户交互。

该属性主要会影响 `onPress` 等交互。文档没有说明禁用状态是否自动产生视觉变化。

### `hidden`

```ts
boolean
```

- 可选。
- 支持 Android、iOS 和 Web。
- 控制组件是否隐藏。

当前文档没有说明隐藏后是否保留布局空间，也没有说明它与条件渲染的具体差异。

### `onPress`

```ts
() => void
```

- 可选。
- 支持 Android、iOS 和 Web。
- 组件被按下时调用。

在 Web 中开发者习惯使用“点击”，而移动端通常使用“按下”描述触摸交互。文档没有展开说明键盘操作、手势冲突或无障碍行为。

### `onAppear`

```ts
() => void
```

- 可选。
- 支持 Android、iOS 和 Web。
- 组件出现在屏幕上时调用。

### `onDisappear`

```ts
() => void
```

- 可选。
- 支持 Android、iOS 和 Web。
- 组件从屏幕移除时调用。

这两个回调属于界面生命周期事件。不要未经验证就把它们等同于 React Web 中某个固定的 `useEffect` 执行时机；当前文档没有说明重复触发规则或平台间时序差异。

### `modifiers`

```ts
ModifierConfig[]
```

- 可选。
- 仅支持 Android 和 iOS。
- 用于传入平台专属 modifier 配置。
- 配置分别来自：
  - `@expo/ui/swift-ui/modifiers`
  - `@expo/ui/jetpack-compose/modifiers`

“逃生舱口”意味着：当通用 API 无法满足需求时，可以使用底层原生平台能力。

使用 `modifiers` 会引入平台差异，Web 端也不支持该属性。因此，共享组件使用它以后，通常需要考虑平台分支以及不同平台的单独测试。

### `testID`

```ts
string
```

- 可选。
- 支持 Android、iOS 和 Web。
- 为组件提供端到端测试定位标识。

它的用途类似 Web 自动化测试中的稳定选择器，能够避免测试依赖易变化的文本或布局结构。

## 注意事项与限制

1. 当前页面针对下一个 Expo SDK 版本，不应直接视为 SDK 56 的稳定 API 保证。
2. `Row` 的统一接口背后是三种不同实现，复杂行为可能存在平台差异。
3. `style` 只支持明确列出的有限属性，不支持任意 CSS 或完整的 React Native 样式。
4. `modifiers` 仅适用于 Android 和 iOS，会降低代码的平台通用性。
5. `spacing` 使用密度无关像素，而不是物理屏幕像素。
6. `alignment` 控制垂直交叉轴，不控制水平方向的内容分布。
7. 文档没有说明换行、溢出、RTL 布局、无障碍属性或动画行为。
8. 文档没有给出开发构建、原生工程配置或额外目录配置要求。

## React Web 开发者的使用思路

可以先建立以下对应关系：

| React Web 思维 | `Row` 中的方式 |
| --- | --- |
| `display: flex; flex-direction: row` | 使用 `<Row>` |
| `gap` | `spacing` |
| `align-items` | `alignment` |
| 中间元素 `flex-grow: 1` | `<Spacer flexible />` |
| `onClick` | `onPress` |
| 测试选择器 | `testID` |
| CSS / style 对象 | 有限的 `style` 属性或原生 `modifiers` |

这些对应关系用于理解概念，不代表 API 和运行机制完全相同。移动端没有浏览器 DOM，Android 与 iOS 也使用各自的原生 UI 系统。

## 实际开发建议

以下属于**基于经验建议**：

- 优先使用 `Row`、`spacing`、`alignment` 和 `Spacer` 完成跨平台布局。
- 只有通用 API 无法表达需求时，再考虑 `modifiers`。
- 使用 `modifiers` 后，应分别验证 Android 和 iOS，并为 Web 提供替代实现。
- 不要把 Web CSS 属性直接迁移到 `style`；先检查 `RowProps` 是否明确支持。
- 对依赖容器宽度的 `Spacer` 布局，应在不同屏幕尺寸下测试。
- 项目使用稳定版 Expo SDK 时，应查阅该 SDK 对应的版本化文档，而不是直接照搬未发布版本示例。

## 文档明确说明与合理推导

### 文档明确说明

- `Row` 是水平布局容器。
- 子元素从起始端向结束端排列。
- Android、iOS 和 Web 使用不同的底层实现。
- `alignment` 控制垂直交叉轴对齐。
- `Spacer flexible` 可以把内容推向相反两端。
- `style` 仅支持指定的平台无关样式。
- `modifiers` 仅支持 Android 和 iOS。
- 组件支持交互、出现、消失和测试定位回调或属性。

### 基于文档内容推导

- `Row` 可近似理解为 React Web 中的水平 Flex 容器。
- `alignment` 的作用近似于水平 Flex 容器中的 `align-items`。
- `Spacer flexible` 需要容器存在剩余宽度才能明显拉开内容。
- 使用 `modifiers` 会增加平台分支和测试成本。
- 由于底层实现不同，跨平台界面不能只在单一平台验证。

## 总结

`Row` 为 `@expo/ui` 组件提供统一的水平布局能力。常规开发主要需要掌握三个配置：

- 使用 `spacing` 设置子元素间距。
- 使用 `alignment` 设置垂直对齐。
- 使用 `Spacer flexible` 分配剩余空间。

它与 Web Flexbox 在概念上相似，但底层由 SwiftUI、Jetpack Compose 和 Web Flex 分别实现。实际使用时，最需要注意的是版本状态、有限的 `style` 属性范围，以及 `modifiers` 带来的平台专属行为。
