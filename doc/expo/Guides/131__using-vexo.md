# 使用 Vexo

> 原文地址：https://docs.expo.dev/guides/using-vexo/

本指南介绍如何在 Expo 项目中安装和配置 Vexo，以实现实时用户分析（Real-time User Analytics）。

---

## 概述

Vexo 是一款**实时用户分析**工具，专为 Expo 项目设计。它可以帮助开发者深入了解用户行为、识别使用中的障碍，并优化用户互动体验。

Vexo 的核心优势在于**极简集成** —— 只需两行代码即可完成基础集成，之后便会自动采集数据。同时，它也支持开发者自定义特定的跟踪行为，满足更精细化的分析需求。

> **术语解释（面向初学者）**：
> - **用户分析（User Analytics）**：收集和分析用户在应用中的行为数据，例如用户点击了哪些页面、停留了多长时间等，从而帮助开发者改进产品。
> - **实时（Real-time）**：数据几乎是即时采集和展示的，无需等待数小时或数天才能看到结果。

---

## 核心功能

Vexo 提供以下主要功能：

| 功能 | 说明 |
|------|------|
| **完整仪表板（Full Dashboard）** | 监控活跃用户数、会话时长、安装量、操作系统分布、应用版本使用情况、地理位置数据以及热门页面等指标 |
| **会话回放（Replays）** | 回放真实用户在应用中的操作过程，直观了解用户的实际操作路径 |
| **热力图（Heatmaps）** | 可视化展示用户点击最频繁的区域，帮助定位界面中的热点 |
| **转化漏斗（Funnels）** | 分析用户的导航路径，找出用户在转化流程中流失的环节，从而提升转化率 |
| **自定义功能（Customization）** | 自定义特定的跟踪事件，并可个性化仪表板的展示方式 |

> **术语解释（面向初学者）**：
> - **会话（Session）**：用户从打开应用到关闭应用的一次完整使用过程。
> - **转化漏斗（Funnel）**：用户完成某个目标（如注册、购买）所经历的步骤序列，帮助分析在哪个步骤用户流失最多。
> - **热力图（Heatmap）**：用颜色深浅表示用户在界面上不同区域的点击频率，颜色越深代表点击越频繁。

## 设置步骤

### 第一步：注册账号并获取 API 密钥

前往 [Vexo 官网](https://www.vexo.co/) 注册账号，然后创建一个新的项目。创建完成后，你将获得一个 **API 密钥（API Key）**，后续集成时需要用到它。

> **术语解释（面向初学者）**：
> - **API 密钥（API Key）**：一个唯一的字符串，用于标识你的项目身份。就像门禁卡一样，只有持有正确的密钥才能向 Vexo 服务器发送数据。

### 第二步：安装依赖包

使用你常用的包管理器安装 `vexo-analytics` 包：

```bash
npm install vexo-analytics
```

```bash
yarn add vexo-analytics
```

```bash
pnpm add vexo-analytics
```

```bash
bun install vexo-analytics
```

> **基于经验建议**：以上四种包管理器（npm、yarn、pnpm、bun）选择你项目中正在使用的那一种即可，无需全部执行。如果不确定当前使用的是哪个包管理器，可以查看项目根目录下是否存在 `yarn.lock`（yarn）、`pnpm-lock.yaml`（pnpm）或 `package-lock.json`（npm）来判断。

### 第三步：在入口文件中初始化 Vexo

在你的主入口文件（通常是 `App.tsx` 或 `index.tsx`）中，导入并调用 Vexo 的初始化函数，传入你的 API 密钥：

```tsx
import { vexo } from 'vexo-analytics';

// 你可能希望使用 `if (!__DEV__) { ... }` 包裹，使 Vexo 仅在生产环境中运行
vexo('YOUR_API_KEY');
```

> **术语解释（面向初学者）**：
> - **入口文件（Entry File）**：应用启动时最先执行的文件，是整个应用的起点。
> - **`__DEV__`**：React Native 内置的全局常量。在开发模式下为 `true`，在生产构建中为 `false`。使用 `if (!__DEV__)` 可以确保分析代码只在生产环境中执行，避免开发调试时产生干扰数据。

> **基于经验建议**：强烈建议使用 `if (!__DEV__)` 条件包裹 Vexo 初始化代码。这样在开发阶段不会产生大量的测试数据污染你的分析仪表板，同时也能减少开发时的网络请求开销。完整写法如下：
>
> ```tsx
> import { vexo } from 'vexo-analytics';
>
> if (!__DEV__) {
>   vexo('YOUR_API_KEY');
> }
> ```

### 第四步：重新编译项目

由于 `vexo-analytics` 库使用了**原生模块（Native Modules）**，安装后必须重新编译项目，而不能仅依赖热重载（Hot Reload）。

```bash
npx expo run:ios
```

```bash
npx expo run:android
```

> **术语解释（面向初学者）**：
> - **原生模块（Native Modules）**：使用平台原生语言（如 iOS 的 Swift/Objective-C、Android 的 Kotlin/Java）编写的代码模块。它们可以直接访问设备底层功能，但需要编译后才能生效。
> - **热重载（Hot Reload）**：在不重新编译整个应用的情况下，实时更新 JavaScript 代码的变更。对于原生模块的变更，热重载无法生效。

### 第五步：验证集成是否成功

重新编译并运行应用后，打开 Vexo 的在线仪表板，检查是否已经开始接收数据。通常数据会在短时间内出现在仪表板中。

## 兼容性说明

| 环境 | 兼容性 | 备注 |
|------|--------|------|
| **自定义开发构建（Custom Development Build）** | 完全兼容 | 无需额外配置插件即可使用 |
| **Expo Go** | 不兼容 | Vexo 依赖自定义原生代码，Expo Go 环境无法支持 |

> **术语解释（面向初学者）**：
> - **自定义开发构建（Custom Development Build）**：使用 `expo-dev-client` 创建的、包含你项目特有原生代码的构建版本。它允许你使用任意原生模块，同时保留 Expo 的开发体验。
> - **Expo Go**：Expo 提供的通用预览应用，可以快速运行大部分 Expo 项目，但无法加载需要自定义原生代码的第三方库。

> **基于文档内容推导**：如果你的项目当前仍在使用 Expo Go，并且希望引入 Vexo，你需要先迁移到自定义开发构建工作流。可以参考 Expo 官方文档中关于"从 Expo Go 迁移到开发构建"的相关指南。

---

## 限制与注意事项

1. **不支持 Expo Go**：由于 Vexo 依赖自定义原生代码，无法在 Expo Go 沙盒环境中运行。
2. **需要原生编译**：每次安装或更新 `vexo-analytics` 包后，都必须重新编译原生代码。
3. **生产环境优先**：建议使用 `__DEV__` 判断来限制 Vexo 仅在生产环境中运行，以免开发调试数据影响分析结果。
