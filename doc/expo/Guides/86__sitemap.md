# Sitemap（站点地图）

> **原始文档地址**：[https://docs.expo.dev/router/reference/sitemap/](https://docs.expo.dev/router/reference/sitemap/)

---

## 概述

本文档介绍如何在 Expo Router 中使用**站点地图（Sitemap）**功能来调试应用的路由。站点地图是 Expo Router 内置的调试工具，它会自动生成一个特殊页面，列出应用中所有的路由（route），帮助开发者快速查看和排查路由配置。

> **关键术语解释**：
>
> - **Sitemap（站点地图）**：在 Web 开发中，站点地图通常是一个列出网站所有页面的文件，用于帮助搜索引擎抓取。在 Expo Router 中，站点地图是一个自动生成的调试页面（路径为 `/_sitemap`），它列出应用中的所有路由，方便开发者在开发阶段查看和调试。
> - **Route（路由）**：路由是应用中每个页面对应的 URL 路径。例如，`/home`、`/settings`、`/form-sheet` 都是路由。Expo Router 采用基于文件系统的路由机制——你在 `app/` 目录下创建的每个文件都会自动成为一个路由。
> - **Deep Link（深度链接）**：深度链接是指直接指向应用内某个特定页面的链接，而不是应用首页。例如，`myapp://profile/123` 可以直接打开应用中的用户资料页面。测试深度链接对于确保用户从外部链接（如邮件、短信、其他应用）能正确跳转到目标页面非常重要。
> - **Expo Go**：Expo 官方提供的开发客户端应用，可以在真实设备上快速预览和测试 Expo 项目，无需构建原生代码。
> - **uri-scheme**：一个命令行工具（CLI），用于在真实设备上测试 URI scheme（统一资源标识符方案），可以模拟从外部打开深度链接的场景。

## 测试深度链接

### 在原生设备上测试

在原生平台（iOS / Android）上，你可以使用 [`uri-scheme`](https://www.npmjs.com/package/uri-scheme) 命令行工具来测试在设备上打开原生链接的效果。

> **关键术语解释**：
>
> - **CLI（Command Line Interface，命令行界面）**：通过终端（Terminal）输入命令来操作的工具，与之相对的是 GUI（图形用户界面）。
> - **URI Scheme（统一资源标识符方案）**：一种定义如何打开特定资源的协议格式。例如，`exp://` 是 Expo 使用的 URI scheme，`https://` 是网页使用的 scheme。

例如，如果你想在 iOS 设备上启动 Expo Go 应用并直接打开 `/form-sheet` 路由，可以运行以下命令：

#### npm

```sh
npx uri-scheme open exp://192.168.87.39:19000/--/form-sheet --ios
```

#### yarn

```sh
yarn dlx uri-scheme open exp://192.168.87.39:19000/--/form-sheet --ios
```

#### pnpm

```sh
pnpm dlx uri-scheme open exp://192.168.87.39:19000/--/form-sheet --ios
```

#### bun

```sh
bunx uri-scheme open exp://192.168.87.39:19000/--/form-sheet --ios
```

> **参数说明**：
>
> | 参数 | 说明 |
> |------|------|
> | `exp://192.168.87.39:19000` | Expo 开发服务器的地址，格式为 `exp://<IP地址>:<端口>` |
> | `/--/` | 分隔符，用于区分开发服务器地址和路由路径 |
> | `form-sheet` | 要打开的目标路由路径 |
> | `--ios` | 指定在 iOS 模拟器或设备上打开（使用 `--android` 则在 Android 上打开） |

> **⚠️ 重要提示**
>
> 请将上面命令中的 `192.168.87.39:19000` 替换为你运行 `npx expo start` 时终端中显示的实际 IP 地址。每次启动开发服务器时，IP 地址可能会不同，这取决于你的网络环境。

> 📌 **基于经验建议**：如果你在使用 VPN 或有多块网卡（例如同时连接了 Wi-Fi 和有线网络），终端显示的 IP 地址可能不是设备能访问到的那个。建议在运行命令前，先确保手机和电脑处于同一局域网下，并确认 IP 地址可以互相 ping 通。

### 在浏览器中测试

你也可以直接在手机浏览器（如 Safari 或 Chrome）中输入链接地址来测试深度链接在真实设备上的表现。

如需了解更多关于测试深度链接的信息，请参阅 React Navigation 官方文档：[Testing Deep Links（测试深度链接）](https://reactnavigation.org/docs/deep-linking)。

> 📌 **基于经验建议**：在浏览器中测试深度链接时，建议在地址栏输入完整的 URL（例如 `https://your-domain.com/profile/123`），观察应用是否能正确响应并跳转到目标页面。这在调试 Universal Links（iOS）和 App Links（Android）时特别有用。

## 站点地图（Sitemap）

Expo Router 会自动注入一个 **`/_sitemap`** 路由，该页面提供应用中所有路由的完整列表。这是一个非常实用的调试工具。

> **关键术语解释**：
>
> - **注入（Inject）**：这里指 Expo Router 在构建过程中自动添加了一个额外的页面（`/_sitemap`），开发者无需手动创建这个页面。它是由框架在后台自动生成的。
> - **`/_sitemap`**：这是一个特殊的路由路径。在开发模式下，你可以在浏览器或 Expo Go 中访问 `http://<你的IP>:<端口>/_sitemap` 来查看所有已注册的路由列表。

### 如何使用站点地图

当你的应用运行在开发模式时，可以直接访问 `/_sitemap` 路径来查看应用中所有的路由。这对于以下场景特别有用：

- **验证路由是否正确注册**：确认你在 `app/` 目录下创建的文件是否被 Expo Router 正确识别为路由。
- **排查路由缺失问题**：如果某个页面无法通过 URL 访问，可以通过站点地图确认该路由是否存在。
- **了解路由结构**：在接手一个已有的项目时，站点地图可以帮助你快速了解应用的整体路由结构。

> 📌 **基于文档内容推导**：由于站点地图列出了应用中所有的路由，它本质上反映了 `app/` 目录下的文件结构。如果站点地图中的路由与你预期的不一致，通常意味着 `app/` 目录中的文件组织存在问题——可能是文件命名不符合约定、目录层级有误，或者某些文件被意外忽略。

## 禁用站点地图

如果你不需要自动生成的站点地图（例如在生产环境中不希望暴露所有路由信息），可以通过修改应用配置文件来禁用它。

在 `app.json`（或 `app.config.js` / `app.config.ts`）中，将 `expo-router` 配置插件的 `sitemap` 属性设置为 `false`：

```json
{
  "plugins": [
    [
      "expo-router",
      {
        "sitemap": false
      }
    ]
  ]
}
```

> **配置说明**：
>
> | 字段 | 类型 | 默认值 | 说明 |
> |------|------|--------|------|
> | `sitemap` | `boolean` | `true` | 设置为 `false` 可禁用自动生成的 `/_sitemap` 路由 |

> **⚠️ 注意事项**
>
> - 禁用站点地图后，`/_sitemap` 路由将不再可用。如果你只是想在生产环境中隐藏它，请注意 Expo Router 默认只在开发模式下注入站点地图，生产构建中通常不会包含。
> - 修改 `app.json` 后，需要重新启动开发服务器（`npx expo start`）才能使配置生效。

> 📌 **基于经验建议**：在开发阶段建议保留站点地图功能，它是一个非常方便的路由调试工具。只有在有特殊安全需求或确实不需要此功能时，才考虑禁用它。如果你使用的是自定义的 `_layout.tsx` 根布局文件，站点地图功能仍然可用——它不受自定义布局的影响。

## 完整示例：项目配置与调试流程

以下是一个典型的使用站点地图进行调试的完整流程示例：

### 第一步：确认项目配置

确保 `app.json` 中已正确配置 `expo-router` 插件：

```json
{
  "expo": {
    "name": "my-app",
    "slug": "my-app",
    "plugins": [
      [
        "expo-router",
        {
          "sitemap": true
        }
      ]
    ]
  }
}
```

### 第二步：启动开发服务器

```sh
npx expo start
```

终端将显示开发服务器的 IP 地址和端口号，例如：

```
Metro waiting on exp://192.168.1.100:8081
```

### 第三步：访问站点地图

在浏览器中访问 `http://192.168.1.100:8081/_sitemap`，你将看到应用中所有路由的列表。

### 第四步：测试深度链接

使用 `uri-scheme` 工具测试特定路由的深度链接：

```sh
npx uri-scheme open exp://192.168.1.100:8081/--/form-sheet --ios
```

> 📌 **基于经验建议**：如果站点地图页面显示为空白或返回 404 错误，请检查以下几点：(1) 开发服务器是否正在运行；(2) 是否在 `app.json` 中意外禁用了站点地图；(3) 是否存在自定义的 `_sitemap.tsx` 文件覆盖了默认行为。

## 常见问题

### 站点地图在生产环境中是否可用？

> 📌 **基于文档内容推导**：根据文档的描述，站点地图是由 Expo Router 在开发阶段自动注入的调试工具。在生产构建中，Expo Router 通常不会注入此页面。如果你希望在生产环境中也禁用开发阶段的站点地图，可以通过上述配置将 `sitemap` 设置为 `false`。

### 我可以自定义站点地图页面的样式吗？

> 📌 **基于文档内容推导**：文档未提及自定义站点地图页面样式的方法。站点地图是 Expo Router 自动注入的内置页面，主要用于调试目的。如果你需要一个自定义的路由列表页面，建议在应用中创建自己的页面来实现，而不是尝试修改内置的 `/_sitemap` 页面。

### 禁用站点地图会影响路由功能吗？

> 📌 **基于文档内容推导**：不会。站点地图仅是一个调试辅助工具，禁用它不会影响应用的实际路由功能。所有在 `app/` 目录下定义的路由仍然会正常工作。
