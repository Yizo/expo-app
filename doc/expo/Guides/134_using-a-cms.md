# 在 Expo 应用中使用 CMS

## 文档解决的问题

CMS（内容管理系统）用于创建、管理和组织博客文章、图片、商品信息等数字内容，无需为内容运营编写定制后台。编辑、市场等非技术人员可以通过管理界面更新内容。

在 Expo/React Native 应用中接入 CMS，可以远程管理内容、快速向用户提供新信息，并在不发布新 App 版本的情况下扩展内容运营。

## 与 React Web 的关系

CMS 的角色与 Headless CMS 在 React Web 中类似：应用负责展示，内容团队通过独立平台维护数据。移动端的重要开发影响是，内容更新可以脱离应用商店发版周期。

“无需发布新 App 更新”指远程内容变化可由应用运行时获取；它不代表 CMS 能替代原生功能或应用代码的发布。这一点是基于文档描述作出的边界解释。

## 文档列出的 CMS

### Strapi

文档提供 Strapi 的 Expo integration 入口，用于了解如何把 Strapi 接入 Expo 应用。

### Sanity

文档提供 Sanity 在 Expo/React Native 中实现 Visual Editing 的指南入口。Visual Editing 强调编辑者在可视化上下文中调整内容。

## 适用场景

- 内容需要由非开发人员频繁更新
- 博客、图片、商品资料等内容不希望与 App 商店发版绑定
- 希望减少定制内容后台的开发成本
- 需要让内容运营规模随业务增长

## 限制、易误解点与建议

- 本文是方案概览，只列出 Strapi 和 Sanity，不代表它们是唯一可用选择，也没有比较二者。
- 当前文档未涉及安装命令、API 请求、鉴权、缓存、离线支持、图片优化、内容模型、实时预览配置或 EAS 设置。
- 当前文档也未说明应用如何发现内容变化；不能据此假定自动推送、实时订阅或后台刷新已经存在。
- **基于文档内容推导**：适合远程化的是内容数据，而需要新增原生能力或改变 App 逻辑的需求仍应走应用构建/更新流程。
- **基于经验建议**：选型时应额外评估 API、权限、离线与缓存能力，但这些并非当前页面明确给出的标准。
