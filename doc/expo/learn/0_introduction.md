## 阅读前需要理解的背景

### React Native

React Native 使用 React 的组件化开发方式构建移动应用。

对于 React Web 开发者，需要先注意一个关键差异：

- React Web 通常使用 `div`、`span`、`img` 等 HTML 元素。
- 示例中的 React Native 使用 `View`、`Text` 等组件。
- 样式通过 `StyleSheet.create()` 定义，而不是直接依赖浏览器中的 CSS 和 DOM。

示例代码中的对应关系可以初步理解为：

| React Native | React Web 中近似的理解 |
|---|---|
| `View` | 类似用于布局的 `div` |
| `Text` | 用于显示文本的组件 |
| `StyleSheet.create()` | 类似集中定义样式对象 |
| `flex`、`alignItems`、`justifyContent` | 与 Web Flexbox 概念相近 |

> **注意：**以上对应关系只用于帮助理解。原文没有说明 React Native 组件与 HTML 元素完全等价。

### Expo

根据原文，这套教程会帮助读者开始使用 Expo，并熟悉 Expo SDK。

可以从教程目标理解为：Expo 提供了一套构建通用应用所需的开发能力，包括访问系统媒体库、配置状态栏、启动屏幕和应用图标等。

> **基于文档内容推导：**Expo 不仅负责运行 React Native 代码，还提供了对设备能力和应用配置的统一开发入口。

### 通用应用

原文所说的 universal app，是指应用使用同一个代码库支持 Android、iOS 和 Web。

这并不代表三个平台不存在差异。教程明确包含“处理 Android、iOS 和 Web 之间的平台差异”。

## 教程核心内容与流程

### 1. 创建启用 TypeScript 的 Expo 应用

教程首先会使用默认模板创建应用，并启用 TypeScript。

对于 React Web 开发者，这类似于使用项目脚手架创建一个预配置的 React 项目，而不是手动搭建全部构建工具。

当前导读页没有提供：

- 创建命令
- Node.js 版本要求
- 包管理器要求
- Expo 项目的具体目录结构

这些内容需要进入下一篇“创建第一个应用”文档学习。

### 2. 使用 Expo Router 创建底部标签页

教程会实现一个包含两个页面的底部标签页布局。

`Expo Router` 用于组织页面及页面之间的导航。这里的“底部标签页”是移动应用中常见的导航形式，用户可以点击屏幕底部的标签切换页面。

> **基于文档内容推导：**对于 React Web 开发者，可以暂时把 Expo Router 理解为应用的路由和导航系统，但移动端的页面切换及底部标签栏并不等同于浏览器中的普通 URL 导航。

当前文档没有说明：

- 两个页面的具体内容
- 路由文件如何命名
- 底部标签页如何配置
- Expo Router 的路由规则

### 3. 使用 Flexbox 实现应用布局

教程会先拆解应用界面的布局结构，再使用 Flexbox 实现。

React Web 开发者通常已经熟悉 Flexbox，但不能直接假设 React Native 的样式能力与浏览器 CSS 完全一致。当前文档只明确说明会使用 Flexbox，没有列出具体差异。

示例中的容器样式为：

```tsx
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
```

各属性在示例中的作用可以理解为：

- `flex: 1`：让容器占用可用空间。
- `backgroundColor: '#fff'`：设置白色背景。
- `alignItems: 'center'`：在交叉轴方向居中子元素。
- `justifyContent: 'center'`：在主轴方向居中子元素。

### 4. 从系统媒体库选择图片

教程会调用各个平台提供的系统界面，让用户从媒体库中选择图片。

这里涉及移动端开发中的设备能力：应用不只是渲染界面，还需要与操作系统提供的功能交互。

当前文档没有说明：

- 使用哪个 Expo SDK API
- 是否需要申请权限
- Android、iOS 和 Web 的具体行为差异
- 用户拒绝权限时如何处理

因此不能仅根据本页确定实际实现方式。

### 5. 创建贴纸选择弹窗

教程会使用 React Native 的两个组件：

- `<Modal>`：显示弹窗或覆盖层。
- `<FlatList>`：渲染列表数据。

它们将用于创建贴纸选择弹窗。

对于 React Web 开发者：

- `<Modal>` 不是 HTML 原生元素，也不是浏览器的普通 DOM 弹窗。
- `<FlatList>` 不是 `Array.map()` 的简单语法替代品，而是 React Native 提供的列表组件。

当前文档没有进一步说明这两个组件的属性、性能特点或限制。

### 6. 添加贴纸触摸手势

教程会增加触摸手势，让用户与贴纸进行交互。

这意味着应用需要处理移动设备上的触摸输入，而不只是 Web 中常见的鼠标点击。

当前文档没有说明：

- 支持哪些手势
- 使用哪个手势库或 API
- Web 上如何处理鼠标或触控板输入
- 手势冲突如何解决

### 7. 使用第三方库截图并保存

教程会使用第三方库：

1. 捕获应用界面截图。
2. 将截图保存到磁盘。

这里不仅涉及 React Native 组件，还涉及文件保存和不同平台的存储能力。

当前文档没有给出：

- 第三方库名称
- 安装命令
- 文件保存路径
- 存储权限要求
- Web 与移动端保存机制的区别

### 8. 处理平台差异

教程会专门处理 Android、iOS 和 Web 之间的差异。

这是理解“单一代码库”的关键：单一代码库不代表所有代码在所有平台都完全相同，而是项目会尽可能共享代码，并在必要时处理平台特有行为。

> **基于文档内容推导：**开发通用应用时，不能只在 Web 环境验证功能。涉及系统媒体库、触摸手势、文件保存和系统界面的功能尤其需要考虑不同平台。

### 9. 完善应用配置

教程最后会配置：

- 状态栏
- 启动屏幕
- 应用图标

这些属于移动应用的系统展示和品牌配置。

对于 React Web 开发者，可以进行如下初步理解：

- 状态栏：设备顶部显示时间、电量等信息的系统区域。
- 启动屏幕：应用启动过程中显示的界面。
- 应用图标：安装后显示在设备桌面或应用列表中的图标。

当前导读页没有提供具体配置文件、字段或资源尺寸要求。

## 示例代码说明

文档提供了一个基础 React Native 组件：

```tsx
import { StyleSheet, Text, View } from 'react-native';

export default function Index() {
  return (
    <View style={styles.container}>
      <Text>Hello world!</Text>
    </View>
  );
}
```

这段代码展示了几个基础概念：

- 组件仍然使用 React 函数组件。
- 页面通过 `export default` 导出。
- 界面不是由 HTML 标签构成，而是由 React Native 组件构成。
- `View` 用于布局。
- `Text` 用于显示文本。
- 样式通过 `style` 属性传递。

教程页面还会使用绿色高亮标记重要代码或相邻示例之间发生变化的代码：

- 桌面端可以将鼠标悬停在高亮位置查看解释。
- 移动端可以点击高亮位置查看解释。

这意味着阅读后续章节时，不应只复制完整代码，还应重点检查高亮区域，理解每一步增加或修改了什么。

## React Web 开发者容易误解的地方

### 单一代码库不等于没有平台差异

教程明确会处理 Android、iOS 和 Web 的平台差异。不能假设所有设备 API 和交互方式都与浏览器一致。

### JSX 不代表底层是 HTML

代码看起来仍然是 JSX，但其中的 `View` 和 `Text` 来自 `react-native`，不是浏览器 DOM 元素。

### Flexbox 熟悉，但实现环境不同

教程会使用 Flexbox，但当前文档没有保证 React Native 支持浏览器中的全部 CSS 属性或完全相同的默认行为。

### 移动端功能可能依赖操作系统

媒体库、触摸手势、截图保存、状态栏和启动屏幕都涉及浏览器之外的平台能力。不能直接套用 React Web 中的文件选择、点击事件或下载链接实现。

### Expo Router 不应直接等同于 Web 路由库

Expo Router 同样负责页面组织和导航，但教程目标包含原生底部标签页，因此实际导航体验会涉及移动端界面结构。
