# 构建 StickerSmash 首页

> 原文更新时间：2026 年 6 月 10 日  
> 本章目标：通过 AI Agent 为 Expo 应用添加底部标签导航、照片展示区域和系统相册选择功能。

## 文档解决的问题

本章将一个基础 Expo 应用逐步改造成 StickerSmash 的首页雏形，完成三项功能：

1. 添加 Home 和 About 两个底部导航标签。
2. 在 Home 页面展示占位照片和操作按钮。
3. 调用手机系统相册，让用户选择照片并替换占位图。

本章采用 AI 辅助开发方式：开发者向 Agent 描述需求，由 Agent 下载资源、安装依赖并修改代码。原文明确指出，完成本章不需要开发者亲自编写代码。

## 阅读前需要理解的背景知识

### Expo

Expo 是围绕 React Native 提供的一套开发工具和库。本章使用了 Expo Router、`expo-image` 和 `expo-image-picker`。

对于 React Web 开发者，可以暂时将其理解为：

- React Native：使用 React 的组件和状态模型开发原生移动应用。
- Expo：提供常用移动端能力和开发工具。
- Expo 库：封装相册、图片显示等移动端功能，减少直接编写 iOS 或 Android 原生代码的需要。

### Expo Router

Expo Router 负责页面导航。本章要求使用它实现底部标签栏，在 Home 和 About 页面之间切换。

它的作用类似 React Web 中的路由库，但底部标签栏是移动应用中常见的导航界面。点击标签通常不是跳转到一个传统网页，而是在应用内部切换页面。

原文没有说明具体的路由目录、文件命名或配置方法，这些内容应由 Agent 根据当前项目完成。

### `expo-image`

`expo-image` 是本章指定的图片显示库，用于在 Home 页面中显示占位图以及用户选择的照片。

可以将它类比为 Web 中的 `<img>`，但它面向 React Native 和移动端图片加载场景。原文没有提供其组件属性或详细用法。

### `expo-image-picker`

`expo-image-picker` 用于打开手机的系统照片库，让用户选择图片。

这与 Web 中的文件选择器目标相似，但移动端还涉及系统权限：应用首次访问照片时，操作系统可能会要求用户授权。

## 实现流程

## 1. 添加底部标签导航

向 Agent 提出以下需求：

- 添加一个底部标签栏。
- 创建 Home 和 About 两个标签。
- Home 显示现有首页。
- About 暂时只显示 `About screen`。
- 使用 Expo Router 管理导航。
- 为两个标签配置符合语义的图标：
  - Home：房屋图标。
  - About：信息圆圈图标。
- 使用统一的深色主题。

### 视觉配置

| UI 部分 | 配置 |
|---|---|
| 页面背景 | `#25292e` |
| 页面头部 | `#25292e` |
| 底部标签栏 | `#25292e` |
| 普通文本 | 白色 |
| 当前选中的标签 | 黄色 `#ffd33d` |

### 预期结果

页面底部出现 Home 和 About 两个标签：

- 当前标签以黄色突出显示。
- 点击另一个标签后切换到对应页面。
- Home 保留当前首页内容。
- About 显示临时文本 `About screen`。

### 开发影响

底部标签导航使应用从单页面结构变为多页面结构。后续功能需要放入正确的页面，而不是继续把所有内容写在同一个入口组件中。

> **基于文档内容推导：** Home 和 About 应分别对应独立页面，并由 Expo Router 的标签导航统一管理。但原文没有规定具体文件结构。

## 2. 添加照片展示区域

下一步为 Home 页面添加主要内容：

- 一张位于页面中央的大照片。
- 照片使用圆角。
- 照片下方纵向排列两个按钮。
- 使用 `expo-image` 显示照片。

### 准备图片资源

Agent 需要完成以下操作：

1. 下载 `sticker-smash-assets.zip` 资源包。
2. 解压资源包。
3. 将图片放入项目的 `assets/images` 目录。
4. 替换目录中所有同名文件。

首页使用资源包中的 `background-image.png` 作为占位照片。资源包还包含后续章节会用到的表情贴纸。

### Agent 无法下载资源时

如果 Agent 无法下载或解压文件，需要手动处理：

1. 下载资源压缩包。
2. 解压文件。
3. 将图片复制到 StickerSmash 项目内的 `assets/images` 目录。
4. 覆盖同名文件。
5. 再让 Agent 构建页面，但从提示词中删除下载资源的要求。

### 首页布局

图片下方纵向排列两个按钮：

| 按钮 | 样式和作用 |
|---|---|
| `Choose a photo` | 主要按钮；黄色边框、白色背景、深色文字，并带有小型图片图标 |
| `Use this photo` | 普通按钮；使用白色文字 |

本章只为 `Choose a photo` 添加实际功能。原文没有说明 `Use this photo` 在本章中的点击行为。

### 预期结果

Home 页面应显示：

- 一张占据页面大部分区域的木质海边栈道照片。
- 带圆角的大图。
- 位于照片下方的两个纵向按钮。

## 3. 从系统相册选择照片

最后添加本章第一个实际交互功能。

Agent 需要：

1. 添加 `expo-image-picker` 库。
2. 为 `Choose a photo` 按钮绑定点击行为。
3. 点击后打开手机的系统照片库。
4. 用户选中照片后，用所选照片替换当前图片。
5. 用户取消选择时，继续显示当前图片。

### 权限流程

第一次打开照片库时，手机会请求访问照片的权限。按照教程流程，需要点击 **Allow（允许）**。

权限来自移动操作系统，而不是普通的 React 页面弹窗。只有获得许可后，应用才能访问用户照片。

### 状态变化

图片展示区域存在两种主要状态：

1. 尚未选择照片时，显示 `background-image.png`。
2. 成功选择照片后，显示用户选择的照片。

取消选择不会恢复占位图，也不会清空图片，而是保留操作前正在显示的图片。

> **基于文档内容推导：** 页面需要保存“当前图片”的状态，并且只在成功选中图片时更新它。原文没有展示具体状态变量或实现代码。

### 预期结果

点击 `Choose a photo` 后：

- 系统照片库打开。
- 首次使用时出现照片访问权限请求。
- 选择照片后，新照片替换当前照片。
- 取消选择后，当前照片保持不变。

## React Web 开发者容易误解的地方

### 移动端导航不等于浏览器页面跳转

Expo Router 仍然负责“路由”，但本章的主要导航形式是应用底部标签栏。用户点击标签后在应用内部切换屏幕，不应默认存在浏览器地址栏、网页刷新或传统 HTML 页面。

### 图片不是通过 HTML `<img>` 显示

React Native 不直接使用 DOM 元素。本章明确要求通过 `expo-image` 显示图片，而不是使用 Web 的 `<img>` 标签。

### 照片库不是普通文件输入框

这里不是创建 `<input type="file">`。`expo-image-picker` 调用的是移动操作系统提供的照片选择界面，并受到系统权限控制。

### 本地资源和用户照片来源不同

`background-image.png` 是随项目提供的静态资源；用户从相册选择的图片则来自设备。两者最终显示在同一位置，但获取方式不同。

> **基于文档内容推导：** 实现时需要让图片组件能够在静态占位资源和用户选择结果之间切换。原文未说明二者的数据格式差异及转换方式。

### Agent 提示词是需求描述，不是配置文件

文档提供的文本用于指导 Agent 完成任务，并不是需要直接放进项目的代码或配置。最终实现是否正确，应以页面行为和视觉结果为准。
