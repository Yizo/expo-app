# 技术文档阅读理解与中文 Markdown 生成

## 任务目标

请阅读我提供的技术文档，并基于原文档内容生成一份中文 Markdown 学习文档。

这不是简单摘要，而是要求你在完整理解原文档的基础上，帮助我系统掌握文档中的知识点、流程、注意事项、限制条件和实际开发影响。

## 我的技术背景

我是一个前端开发者，主要开发过 React Web 项目。

但是我没有开发过：

- React Native
- Expo
- 移动端原生开发
- iOS / Android 原生工程

因此，你在解释文档时，不能默认我已经理解 React Native、Expo 或移动端原生开发相关概念。

文档中凡是涉及这些领域的概念、术语、配置、流程、限制和注意事项，都需要用 React Web 开发者能够理解的方式解释清楚。

## 阅读和整理原则

请严格基于当前文档内容进行整理。

你需要先理解原文档的结构和意图，再决定最终 Markdown 文档的组织方式。

不要强行套用固定模板。

如果原文档是教程型文档，就按教程流程整理。

如果原文档是概念型文档，就按概念关系整理。

如果原文档是 API / 配置说明文档，就按配置项、参数、用法和注意事项整理。

如果原文档是故障排查或迁移文档，就按问题、原因、解决方案和风险整理。

最终输出结构应当服务于当前文档，而不是套用固定目录。

## 必须覆盖的内容

无论文档结构如何变化，都必须覆盖以下信息：

1. 当前文档主要解决什么问题
2. 这篇文档适合什么场景
3. 文档中的核心概念
4. 文档中的关键流程
5. 文档中的配置项、命令、文件或目录说明
6. 文档中的注意事项、限制条件、警告和容易踩坑的地方
7. 对 React Web 开发者来说，哪些地方最容易误解
8. 实际开发中应该如何使用这些知识
9. 哪些内容是文档明确说明的
10. 哪些内容是你基于文档内容推导出来的

如果某一类内容在当前文档中不存在，请明确说明“当前文档未涉及”，不要编造。

## 输出要求

请使用中文输出，并生成 Markdown 格式文档。

输出时请遵守以下要求：

- 保留原文档的主要逻辑顺序
- 不要遗漏原文档中的重要知识点
- 不要遗漏注意事项、限制条件和警告
- 不要只翻译，要解释含义和开发影响
- 不要默认我懂 React Native 或 Expo
- 对关键术语要补充解释
- 对命令和配置要说明作用，而不是只列出来
- 对容易误解的地方要重点说明
- 对文档没有明确说明的内容，不要猜测
- 如果给出经验性建议，请标注“基于经验建议”
- 如果是根据文档内容推导出来的结论，请标注“基于文档内容推导”

## 推荐输出方式

请根据文档实际内容，自行设计 Markdown 结构。

但建议至少包含以下几类信息：

```md
# 文档标题

## 文档解决的问题

## 阅读前需要理解的背景知识

## 按原文结构整理的核心内容

## 关键概念解释

## 关键流程 / 命令 / 配置说明

## 注意事项、限制条件和坑点

## React Web 开发者需要特别注意的地方

## 实际开发建议

## 总结
```

注意：以上只是推荐结构，不是强制模板。
如果当前文档不适合这个结构，请按照原文档本身的结构重新组织。

## 重点要求

请优先保证：

1. 忠实于原文档
2. 不遗漏重要信息
3. 对初学 React Native / Expo 的 React Web 开发者友好
4. 结构根据文档内容自然生成
5. 注意事项和限制条件必须重点提取
